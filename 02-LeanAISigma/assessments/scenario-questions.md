## How to Self-Assess

### Domain Question Banks
1. Work through each domain assessment file without looking at answers.
2. Score yourself after completing all questions in a domain.
3. **Target score:** 70% or higher per domain. If you score below 70%, review the corresponding curriculum module before proceeding to exam scenarios.
4. Pay attention to which task statements you miss -- these indicate specific concept gaps, not general domain weakness.

### Exam Scenarios
1. Read the full scenario context before attempting any questions.
2. Answer all three questions for a scenario in sequence, as they build on the same situation.
3. **Target score:** 75% or higher across all scenario questions.
4. After grading, review explanations for both correct and incorrect answers -- the explanations teach reasoning patterns that transfer to other questions.

### Study Recommendations
- **Start with Domain 2 (CORPS Framework, 25% weight)** -- it is the highest-weighted domain and foundational to all other domains.
- **Domains 3, 4, and 5 (20% each)** form the technical core. Study them after Domain 2 since they build on CORPS concepts.
- **Domain 1 (Lean Principles, 15%)** provides the philosophical foundation. Review it first for conceptual grounding, but allocate more study time to higher-weighted domains.
- **Use the cross-reference map** (`docs/cross-reference-map.md`) to trace missed questions back to specific curriculum sections.
- **Simulate exam conditions:** Set a timer for 90 minutes and work through all scenario questions without reference materials. This builds the time management skills needed for the actual assessment.
